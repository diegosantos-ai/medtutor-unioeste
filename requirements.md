# requirements.md

## Objetivo
Organizar o projeto MedTutor-Unioeste para garantir clareza, escalabilidade e facilidade de manutenção.

## Requisitos Funcionais
- Plataforma de tutoria médica com chat, dashboard, recursos de estudo e acompanhamento de progresso.
- Backend em Python (FastAPI), frontend em React/TypeScript.
- Integração com serviços de IA (Gemini, OpenAI).
- Infraestrutura Docker e scripts de automação.

## Requisitos Não Funcionais
- Separação clara entre backend, frontend e documentação.
- Deploy automatizado via Docker Compose.
- Testes automatizados e rastreamento de progresso.
- Documentação acessível e atualizada.

## Restrições
- Não misturar arquivos de backend e frontend.
- Manter scripts e templates em pastas dedicadas.
- Usar padrões de arquitetura definidos por Kiro.

## Critérios de Aceitação
- Estrutura de pastas reorganizada.
- Documentação dos 3 pilares criada e aprovada.
- Progresso rastreado em `progresso.md`.
